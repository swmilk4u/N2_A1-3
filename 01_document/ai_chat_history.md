# AI 코딩 도구 사용 과정 증빙서 (AI Chat History Log)

본 웹 서비스는 AI 코딩 도구(Antigravity)와의 협업을 통해 기획부터 코딩, 트러블슈팅 및 기능 고도화까지 페어 프로그래밍 방식으로 완성되었습니다. 이하 대화 및 문제 해결 흐름을 요약 정리합니다.

---

## 💬 1단계: 기획 및 초기 환경 구성
* **요구사항 분석**: 
  * "이직, 커리어, 창업" 테마에 부합하는 웹 서비스 기획 수립.
  * 순수 바닐라 JS/HTML/CSS 기술 조건에 맞춰 프로젝트 디렉토리 트리 설계.
  * Vercel Serverless Functions(Python 3.12 런타임) 스펙에 부합하도록 최상위 `api/` 경로와 `BaseHTTPRequestHandler` 기반 백엔드 아키텍처 정립.

---

## 🛠️ 2단계: Vercel 빌드 장애 트러블슈팅 (Debugging)
* **발생 오류 1 (`ModuleNotFoundError: google-genai`)**:
  * Vercel 배포 가동 시 백엔드 모듈 빌드 에러(500)가 발생하여 분석이 중단됨.
  * **원인 진단**: Vercel의 Python 컴파일러는 프로젝트 **최상위 루트 디렉토리**의 `requirements.txt`만 감지하여 의존성을 설치함. 기존 코드는 `02_source/requirements.txt`에만 작성되어 빌드 타임에 누락됨.
  * **해결 조치**: 루트 디렉토리에 `google-genai` 패키지 명세를 기재한 `requirements.txt`를 신규 선언하여 자동 배포 에러를 완벽히 해결함.
* **발생 오류 2 (Vercel Runtime Mismatch)**:
  * Vercel Python 런타임 버전 설정 규약이 구버전 형식(`@latest`)으로 되어 배포가 거절됨.
  * **해결 조치**: `vercel.json`을 걷어내고 순수 파이썬 서버리스 디렉토리 감지 빌드 엔진을 타게 유도하여 빌드 프로세스 정상화 완료.

## 🎨 3단계: 사용자 피드백 반영 및 기능 고도화 (Refactoring)
* **🎲 예시 랜덤입력 편의 기능 도입**:
  * 매번 긴 자소서 폼을 수동 작성하기 불편하다는 의견에 따라 프론트에 주사위 버튼 배치.
  * 클릭할 때마다 4종 가상 직군(프론트엔드/백엔드/데이터엔지니어/디자이너)의 다채로운 경험 샘플이 인풋 필드에 동적으로 자동 입력되되 수동 수정이 가능하도록 연동함.
* **🌐 실시간 RSS 뉴스룸 및 디지털시계 구현**:
  * 메인 우측의 정적 비주얼 영역을 실제 커리어 트렌드 습득에 도움이 되는 글래스모피즘 뉴스룸 카드로 대체.
  * CORS 제약을 피하기 위해 백엔드에 `/api/news` 엔드포인트를 구현하여 구글 뉴스 RSS 피드를 프록시 서빙함.
  * 구글 뉴스의 무작위 정렬 및 과거 기사 출력 맹점을 극복하고자 검색 쿼리에 `OR` 연산자와 시간 제한 연산자 **`when:14d`**를 주입하여 100% 2주일 이내 최신 뉴스만 뽑아내도록 정밀화함.
  * 날짜 내림차순 정렬 로직 및 1초 주기로 자동 구동되는 헤더 디지털시계를 탑재하여 비주얼과 동적 UX를 극대화함.

---

## 🧪 4단계: UX 예외 처리 검증
* **빈 값 전송 제어**: 필수값 누락 시 Toast 알림 배너 팝업 및 전송 차단.
* **통신 장애 제어**: 백엔드 오류(500) 발생 시 적색 경고 알림 바 노출.
* **타임아웃(Timeout) 제어**: 60초 초과 지연 시 `AbortController`를 가동하여 브라우저 강제 중단 및 안내 팝업 작동.
